# クローンから自分用環境をつくる

GitHub からコードを取得し、**あなた専用の Convex・Stripe・Vercel** で卓注文アプリを動かすための手順です。
公開URLの取得やテスト決済までの詳細は [README.md](README.md) に続きます。

- 想定: プログラミング未経験でも、上から順に進められる
- 方針: **環境変数はすべて手作業で入力**（コピペ元は自分が発行したキーのみ）
- 所要時間: クローン〜初回公開まで 約 40〜60 分（[README.md](README.md) 全体）

## 全体像

```
 GitHub（配布リポ）
      │  git clone
      ▼
 あなたのPC（コード）
      │  npm install
      │  npx convex dev --configure new  →  .env.local が自動生成
      │  Convex ダッシュボードで環境変数を手入力
      │  vercel link / ./scripts/deploy.sh
      ▼
 Convex（あなたのプロジェクト）  ←── Stripe テストキー（手入力）
      │
      ▼
 Vercel（あなたの公開URL）  →  ブラウザで開く
```

- Mac: アプリ「ターミナル」

## ステップ1：GitHub からクローンする

### 1-1. 作業フォルダを決める

例: デスクトップの `開発` フォルダ。以降、そこでターミナルを開きます。

### 1-2. リポジトリを取得する

```bash
git clone https://github.com/onodemo01/app-convex-gaslab-order.git
cd app-convex-gaslab-order
```

- フォルダ名は `gaslab-order` などにリネームしても構いません（手順書内のパス表記は `app-convex-gaslab-order` でも同じです）。
- `Cloning into...` のあとエラー（`fatal:`）が出なければ成功です。

### 1-3. 中身を確認する

```bash
ls
```

`package.json`・`convex`・`src`・`docs` などが見えればOKです。

> ⚠️ **やってはいけないこと**
> - 講師や他人の `.env.local` をもらってコピーしない（あなたの Convex とは別物になる）
> - デモサイト用の Stripe キーや Convex の画面をそのまま流用しない

## ステップ2：依存パッケージを入れる

プロジェクトフォルダの中で:

```bash
npm install
```

最後に赤い `ERR!` がなければ成功。[README のステップ0](README.md#ステップ0プロジェクトを動かせる状態にする) と同じ作業です。

## ステップ3：自分の Convex プロジェクトを作る

ここで **あなた専用のクラウド上バックエンド** ができます。講師のデモ環境とは完全に別です。

```bash
npx convex login
```

ブラウザで GitHub または Google でログイン → `Approve`。

```bash
npx convex dev --configure new --team <あなたのチーム名> --project gaslab-order
```

- **チーム名**: Convex ダッシュボードに表示されている自分のチーム（1人なら1つだけのはず）
- **リージョン**: `US East (N. Virginia)` を選ぶ（Enter で確定）
- `Convex functions ready!` と出たら **Ctrl+C** で止める

この操作で **`.env.local` が自動作成**されます（Convex の接続先 URL など）。**手で編集する必要は基本ありません。**

本番用デプロイを1回:

```bash
npx convex deploy
```

詳細・スクショ位置は [README ステップ1](README.md#ステップ1convex頭脳をクラウドに置く) を参照。

## ステップ4：環境変数を手入力する（重要）

公開アプリが使うのは Convex の **Production（本番）** 側です。
**Convex ダッシュボード → 左上のデプロイ切替で「Production」を選ぶ → Settings → Environment Variables** から、次の表どおり **1つずつ手で追加**してください。

> ⚠️ `npx convex env set` コマンドでも設定できますが、この手順書では **ダッシュボードでの手入力**を標準とします（入力ミスを目で確認しやすいため）。

### 必須（最低限これだけで公開〜会計開始まで）

| 名前 | 値の入手元 | 入力タイミング | 例 |
|------|------------|----------------|-----|
| `STRIPE_SECRET_KEY` | Stripe ダッシュボード → 開発者 → API キー → **シークレットキー**（`sk_test_` で始まる） | ステップ4で入力 | `sk_test_...` |
| `APP_BASE_URL` | 最初は仮でよい。Vercel 公開後に **あなたの URL** に直す | 仮: `http://localhost:3000` → 公開後に `https://あなた.vercel.app` | `https://my-shop.vercel.app` |

Stripe のキー取得の詳細: [README ステップ2](README.md#ステップ2stripeテスト決済を接続する)

`APP_BASE_URL` の更新タイミング: [README ステップ3-4](README.md#3-4-app_base_url-を本物のurlに直す)

### 任意（余裕があれば・機能ごと）

| 名前 | 用途 | 入手元 |
|------|------|--------|
| `STRIPE_WEBHOOK_SECRET` | 決済完了を自動反映（会計済みまで進む） | Stripe で Webhook 作成後の `whsec_...` → [付録A](README.md#付録a-stripe-の-webhook会計を最後まで自動で通す) |
| `RESEND_API_KEY` | 領収メール送信 | <https://resend.com> の API キー → [付録B](README.md#付録b会計メールresendデフォルトではオフ) |
| `RESEND_FROM` | メール送信元アドレス | Resend で設定した送信元 |
| `ANTHROPIC_API_KEY` | 分析画面の AI 要約 | Anthropic の API キー（未設定でもアプリは動く） |
| `ANTHROPIC_MODEL` | AI モデル名（省略可） | 省略時は `claude-haiku-4-5` |

### 入力しないもの（自動 or 不要）

| 項目 | 理由 |
|------|------|
| `.env.local` の `VITE_CONVEX_URL` など | ステップ3の `convex dev --configure new` で自動生成 |
| Vercel の環境変数（標準手順） | `./scripts/deploy.sh` はプレビルド方式のため **CONVEX_DEPLOY_KEY 不要** |
| 講師デモのキー・URL | 他人の環境と混ざるため使用禁止 |

### 入力後の確認

Convex ダッシュボード（**Production**）の Environment Variables に、少なくとも次の2行があること:

- `STRIPE_SECRET_KEY`
- `APP_BASE_URL`

値は他人に見せない。スクショを撮る場合は `sk_test_` / `whsec_` 部分を隠す。

Stripe と Convex の値の渡し方の図: [stripe-convex-フロー.md](stripe-convex-フロー.md)

## ステップ5：Vercel に公開する

自分の Vercel プロジェクトとしてリンクし、デプロイします。

```bash
npm install -g vercel
vercel login
vercel link
```

プロジェクト名は `gaslab-order` など自由。詳細は [README ステップ3](README.md#ステップ3vercel-に画面を公開する)。

公開:

```bash
npx convex login   # 未ログインなら
./scripts/deploy.sh
```

`✅ デプロイ完了: https://〇〇.vercel.app` と出た URL が **あなたのアプリ**です。

### 5-1. APP_BASE_URL を手で直す（忘れずに）

ステップ4で仮入力した `APP_BASE_URL` を、いま出た Vercel の URL に **ダッシュボードから手で書き換え**ます。

- 末尾に `/` は付けない
- 例: `https://my-shop.vercel.app`

1. 公開 URL をブラウザで開く

## うまくいかないとき

| 症状 | 確認すること |
|------|----------------|
| `git clone` で失敗 | ネット接続、URL の typo、社内プロキシ |
| `npm install` で赤エラー | プロジェクトフォルダの中で実行しているか、`node -v` が v18 以上か |
| 入口は出るが「店舗未設定」 | 公開 URL で「デモデータ投入」を押したか |
| 会計で「開始に失敗」 | Convex **Production** に `STRIPE_SECRET_KEY` があるか、値が `sk_test_` か |
| 決済後に会計済みにならない | 正常（Webhook 未設定）。付録Aで `STRIPE_WEBHOOK_SECRET` を手入力 |
| デプロイ失敗 | `npx convex login`・`vercel link` 済みか、`./scripts/deploy.sh` をプロジェクト直下で実行したか |

その他: [README のトラブルシュート](README.md#うまくいかないときよくあるつまずき)
