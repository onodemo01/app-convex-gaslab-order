# クローンから自分用環境をつくる（ステップ1〜2）

GitHub にあるコードを取ってきて、**あなた専用の Convex・Stripe・Vercel** で
卓注文アプリを動かすための、いちばん最初の手順です。

> 🔰 **この資料のゴール**
> 「ネット上のコードを自分のパソコンに取り込み、動かす準備（部品のインストール）まで」終わらせる。
> ここまでで**全6ステップのうち 1〜2 が完了**します。続き（ステップ3〜6）は [README.md](README.md) へ。

- **むずかしさ**：コマンドを2〜3回コピペするだけ。プログラミングの知識は不要です
- **かかる時間**：この資料は 10〜15 分（全体＝クローン〜公開で 40〜60 分）
- **お金**：かかりません（すべて無料枠・テストモード）

## 🧰 用意するもの

| 項目 | 内容 | かんたんに言うと |
|------|------|----------------|
| PC | Mac または Windows＋ネット | 手元のパソコン |
| Node.js | **v20 LTS 推奨**（`node -v` で確認。無ければ <https://nodejs.org/> の LTS） | アプリを動かすエンジン |
| Git | クローン用。Mac は Xcode Command Line Tools、Windows は <https://git-scm.com/> | コードを取ってくる道具 |
| GitHub アカウント | clone は**不要**（公開リポ）。Convex / Vercel ログインに**あると便利** | ログイン用のID |
| Convex | <https://convex.dev> で無料登録 | データの置き場（頭脳） |
| Stripe | <https://stripe.com/jp> で無料登録（**テストモード**だけ使う） | 支払いの仕組み |
| Vercel | <https://vercel.com> で無料登録 | 画面をネットに出す場所 |

> 🖥 **「ターミナル」って？** コマンド（文字の命令）を打ちこむ黒い画面です。
> - **Mac**：アプリ「ターミナル」を開く
> - **Windows**：**Git Bash**（Git for Windows に付いてくる）を開く
>   - ⚠️ `./scripts/deploy.sh` は **PowerShell では動きません**。必ず Git Bash で。

## ステップ1 ／ 6：GitHub からコードを取ってくる

> 🎯 **このステップのゴール**：コードが自分のPCのフォルダに入っている状態。

### 1-1. 作業する場所を決める

例：デスクトップの `開発` フォルダ。以降、**そこでターミナルを開いて**進めます。

### 1-2. コードをコピーする（clone）

ターミナルに、次の2行をコピペして実行します。

```bash
git clone https://github.com/onodemo01/app-convex-gaslab-order.git
cd app-convex-gaslab-order
```

> 📖 `git clone` ＝ ネット上のコードを自分のPCにまるごとコピー。
> `cd` ＝ そのフォルダの中に入る（以降のコマンドはこの中で打ちます）。

- フォルダ名は `gaslab-order` などにリネームしてもOK。
- `Cloning into...` のあとに **赤い `fatal:` が出なければ成功**です。

### 1-3. 中身を確認する

```bash
ls
```

`package.json`・`convex`・`src`・`docs` などが並んで見えれば OK です。
（`ls` ＝ フォルダの中身を一覧表示するコマンド）

> ⚠️ **やってはいけないこと**
> - 講師や他人の `.env.local` をもらってコピーしない（あなたの Convex とは別物になり、動きません）
> - デモサイト用の Stripe キーや Convex 画面をそのまま流用しない
> - **APIキー（`sk_test_...` など）を AI チャットに貼らない**

<details>
<summary>（任意）Cursor など AI エージェントに手伝ってもらう場合</summary>

構築作業のコマンド実行は AI に任せてよいです。ただし**秘密情報だけは人が手入力**します。

| AI に任せてよい | 人がやる（AI に渡さない） |
|---|---|
| `gh` / `npx convex` / `stripe` / `vercel` のコマンド実行 | ブラウザ認証の Approve（承認ボタン） |
| エラーの切り分け・手順の続行 | `STRIPE_SECRET_KEY`・`APP_BASE_URL` の入力 |

</details>

## ステップ2 ／ 6：アプリの部品をインストールする

> 🎯 **このステップのゴール**：`npm install` が最後まで終わっている状態。

プロジェクトフォルダの中（＝ステップ1-2 で `cd` した場所）で：

```bash
npm install
```

> 📖 `npm install` ＝ アプリが動くのに必要な部品（ライブラリ）をネットから集めてくる作業。
> 数分かかることがあります。黄色い `warn` は無視してOK。**最後に赤い `ERR!` が無ければ成功**です。

これで**ステップ1〜2が完了**しました。おつかれさまです。

## 🆘 うまくいかないとき

| こんな症状 | まず確認すること |
|------|----------------|
| `git clone` で失敗する | ネット接続／URL の打ち間違い／会社のプロキシ設定 |
| `git` が「見つからない」と出る | Git が未インストール。上の「用意するもの」から入れる |
| `npm install` で赤い `ERR!` | **フォルダの中で実行しているか**（`cd` したか）／`node -v` が v18 以上か |
| `node` が「見つからない」と出る | Node.js が未インストール。<https://nodejs.org/> の LTS を入れる |
| 入口は出るが「店舗未設定」 | これはステップ6の話。README で「デモデータ投入」を押す |
| 会計で「開始に失敗」 | これはステップ4の話。README で `STRIPE_SECRET_KEY` を**本番**に入れる |

さらに詳しくは [README の「うまくいかないとき」](README.md) を見てください。
