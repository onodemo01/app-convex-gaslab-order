# クローンから自分用環境をつくる

GitHub からコードを取得し、**あなた専用の Convex・Stripe・Vercel** で卓注文アプリを動かすための手順です。
公開URLの取得やテスト決済までの詳細は [README.md](README.md) に続きます。

- 想定: プログラミング未経験でも、上から順に進められる
- 方針: **環境変数はすべて手作業で入力**（コピペ元は自分が発行したキーのみ）
- 所要時間: クローン〜初回公開まで 約 40〜60 分（[README.md](README.md) 全体）

---

## 有料特典リポジトリについて

この手順書が入っている GitHub リポジトリは、セミナー**有料特典**です。

| 項目 | 内容 |
|------|------|
| 公開設定 | **public**（GitHub アカウントがなくても `git clone` 可能） |
| URL の案内 | **有料参加者にのみ**（Peatix / connpass の特典欄・当日案内） |
| リポ URL | `https://github.com/onodemo01/app-convex-gaslab-order` |

- セミナー**当日**に clone する必要はありません。**セミナー後**にこの資料から進めてください。
- URL を SNS 等で広く共有しないでください（有料特典です）。

## 全体像

```
 GitHub（配布リポ）
      │  git clone
      ▼
 あなたのPC（コード）
      │  npm install
      │  npx convex dev --configure new  →  .env.local が自動生成
      │  Convex ダッシュボードで環境変数を手入力
      │  vercel link / ./scripts/deploy.sh
      ▼
 Convex（あなたのプロジェクト）  ←── Stripe テストキー（手入力）
      │
      ▼
 Vercel（あなたの公開URL）  →  ブラウザで開く
```

## 用意するもの

| 項目 | 内容 |
|------|------|
| PC | Mac または Windows、インターネット接続 |
| Node.js | **v20 LTS 推奨**（`node -v` で確認。v18 でも動く場合あり）。無ければ <https://nodejs.org/> の LTS |
| Git | クローン用。Mac は Xcode Command Line Tools、Windows は <https://git-scm.com/> |
| GitHub アカウント | clone は**不要**（public リポ）。Convex / Vercel ログイン用に**推奨** |
| Convex | <https://convex.dev> で無料登録 |
| Stripe | <https://stripe.com/jp> で無料登録（**テストモード**のみ使用） |
| Vercel | <https://vercel.com> で無料登録 |

ターミナルの開き方:
- Mac: アプリ「ターミナル」
- Windows: **Git Bash**（Git for Windows に付属）
  - `./scripts/deploy.sh` は PowerShell では動きません

## ステップ1：GitHub からクローンする

### 1-1. 作業フォルダを決める

例: デスクトップの `開発` フォルダ。以降、そこでターミナルを開きます。

### 1-2. リポジトリを取得する

```bash
git clone https://github.com/onodemo01/app-convex-gaslab-order.git
cd app-convex-gaslab-order
```

- フォルダ名は `gaslab-order` などにリネームしても構いません（手順書内のパス表記は `app-convex-gaslab-order` でも同じです）。
- `Cloning into...` のあとエラー（`fatal:`）が出なければ成功です。

### 1-3. 中身を確認する

```bash
ls
```

`package.json`・`convex`・`src`・`docs` などが見えればOKです。

> ⚠️ **やってはいけないこと**
> - 講師や他人の `.env.local` をもらってコピーしない（あなたの Convex とは別物になる）
> - デモサイト用の Stripe キーや Convex の画面をそのまま流用しない

---

## ステップ2：依存パッケージを入れる

プロジェクトフォルダの中で:

```bash
npm install
```

最後に赤い `ERR!` がなければ成功。ここまでで統一ステップ2まで完了です。

## うまくいかないとき

| 症状 | 確認すること |
|------|----------------|
| `git clone` で失敗 | ネット接続、URL の typo、社内プロキシ |
| `npm install` で赤エラー | プロジェクトフォルダの中で実行しているか、`node -v` が v18 以上か |
| 入口は出るが「店舗未設定」 | README のステップ6で「デモデータ投入」を押したか |
| 会計で「開始に失敗」 | README のステップ4で `STRIPE_SECRET_KEY` を Production に設定できているか |
| デプロイ失敗 | README のステップ5を上から順に再実行 |

その他: [README のトラブルシュート](README.md#うまくいかないときよくあるつまずき)
